windows 10 linux子系統: 如何存取windows中的資料

從昨天我的windows 10更新後，我就一直在研究這項功能的優缺點 當然讓我發現了一件事: 檔案存取 windows 10中linux似乎和windows是在兩個不同的…環境?(我不知道我在說啥 在linux環境中，windows的磁碟機是以掛載…阿不對，是資料夾的形式存在 為甚麼我會這樣說呢?請看下圖 我們可以看見，我的windows中有3個磁碟機，分別為C D T 分別對應至linux環境中 /mnt 底下的三個 c d t 資料夾 我們輸入df -h觀察該資料夾是否是對應/dev底下任何一個檔案 沒有 ls /dev 也沒有任何發現 因此推導出一個結論: 磁碟機是以資料夾的型態出現在linux環境中

讀取測試

首先我們在windows C:\ 底下建立一個文字檔 benny.txt 隨便輸入些內容 然後在linux環境下，輸入cat /mnt/c/benny.txt 測試成功!

寫入測試

在linux環境下輸入nano /mnt/c/benny.txt 編輯並存檔試試(我不喜歡vi，真的) 唉呀!失敗了 為甚麼呢? 因為C:\是受寫入保護的，所以linux環境無法對其寫入(很安全的設計) 那我們試試 D:\ 吧 輸入nano /mnt/d/benny.txt 成功了!! 以上就是全部的說明了 由於這是新功能，很多東西我還沒摸透，如果有高人看到這篇有錯，還請不要吝嗇多指教

我的比特幣大小事

我的比特幣(bitcoin)大小事 - 比特幣介紹 - Part1 - 比特幣概述、挖礦概述(簡單教學)

知道比特幣嗎? <–謎之聲:混網路的都知道啦

它是一種網路貨幣，原文名叫「Bitcoin」，中文名叫「比特幣」(譯音)，也有人叫「位元幣」

就我所知，他有以下特性: <–謎之聲:從「就我所知」這句話來說，很明顯又是一篇沒路用的廢文一篇了

                                    任何交易紀錄都是公開的

                                    你的比特幣錢包(帳戶)絕對不會凍結

                                    交易過程不經過任何類似銀行的機構(點對點交易)

                                   可以在任何國家使用(那是官方說的，但某些國家或公司將比特幣視為不允許或違法的)

                                   開放原始碼(Open source)

如果還是不懂的話，請看以下影片(英文):

讓我們開始過著有比特幣的生活吧!

首先，我們要先有一個錢包，這樣才可以裝錢啊

基本上比特幣的官網都有寫了，就自己挑一個吧:

https://bitcoin.org/zh_TW/choose-your-wallet

錢包安裝(註冊)完了嗎?

接下來就是要教你如何獲取比特幣喔!

要獲得比特幣有以下的方法:

                                      去挖礦(本文介紹)

                                      去 乞討(誤 某些網站領取(待續…)

                                      去賭博(別上癮阿(待續…)

                                      直接去買(比特幣也是貨幣的一種，所以可以玩匯率(但現在它的匯率很不穩定，曾經我看過台幣匯率差到10000元)，它也可以投資)

                                      其他我不知道的方式…

那來一個個解說吧…

1.挖礦

   在此建議大家，除非你是職業級的(電腦運算力高的意思)，不然建議你別走這條路來獲取比特幣

   不過秉持著不專業但獨家的精神的小恩我，就稍微介紹一下

   首先，我們要先知道比特幣的整個交易過程

   當我們把比特幣寄送出去時，原則上來說是直接到目的地

   但是!在這些數據中，帶有你的數位簽證，來證明這些比特幣是你發送的，並且在此同時，你的交易紀錄會寫入一個叫「區塊鏈」的地方。在這裡，所有人都可以看到妳的交易

   挖礦的工作，是在承認整個交易，一般來說，只要有六個礦工去確認，整個交易就算完成了，至於交易速度呢… 這就得看你設定了多少「手續費」

   所以，為了獎勵那些礦工們，比特幣當初在設計演算法時在區塊鏈中加入了「區塊獎勵」

   當你的電腦處理完一個區塊(Block)時，區塊裡面會有一些少許的比特幣當作鼓勵，但是，因為比特幣演算法的規定，每處理完的一個區塊，下一個區塊的獎勵會越來越少(比特幣的發行總量只有2000萬)；以及現在挖礦的競爭越來越激烈了(從以前的顯卡到現在超強大的ASIC礦機)，導致現在的礦工主要以「手續費」維生，千萬別抱著玩玩的心態來挖礦。

   好了，我囉嗦完了(這麼說能不能阻止你，我就不知道了)

   那我們開始吧!

   首先，我們要先有個 甜蜜的歸屬 礦池，在下我使用的是Deepbit，去註冊一支會員並且登入吧!

 圖片好像有點太大了,sorry

來解說一下吧!

1.上面的藍字是指你賺取的比特幣；下面的綠字則是你的總運算力

2.第一個較長的文字框，用來填你的比特幣位址；至於下面的文字框呢，是當你所賺取的比特幣到文字框的時候，況池會自動支付比特幣到你的位址

3.這格區域，是你的礦工(挖礦設備，有時指用挖礦設備挖礦的人)。「Tail」是你的礦工帳號，到時候再操作挖礦程式會用到；「貢獻」是礦池會依照你的的貢獻直來分發給你比特幣；「過期」是指有時候你的貢獻會變得沒用的意思；「Last」指礦池上一次偵測到你的設備有運轉的時間距離現在有多久(謎之聲:從這裡來看，小恩沒挖礦很久了)；「Rate, MH/s」是你的是設備個別的運算力。準備好了嗎?讓我們按「Create new worker」來開始吧

1.綠字參照上一張圖的「貢獻」；紅字參照上一張圖的「過期」

2.第一個文字框，是礦工名稱，名字隨便你取(英文及數字)；第二格是礦工密碼，建議別使用礦池登入密碼(因為有的挖礦程式密碼是用明碼儲存的)

3.是領取方式，「按每貢獻」是指自己挖自己賺，有點獨自挖礦的fu，但這你要保證你的運算力要很高，不然別選這個；「按比例」是指大家的運算力集結起來，再依照你的貢獻去分，但這種收入多半不穩定

好了，開啟你的挖礦軟體吧

本人採用的是有介面(GUI)的挖礦軟體GUIMINER，下載好了就開啟它吧

紅色框裡面的下拉式選單，就選deepbit(今天用的礦池)；「Email」填你的礦工帳號，在不同礦池中礦工帳號格式也不一樣，deepbit的礦工帳號格式是[註冊時的Email]_[你的礦工帳號]

密碼就填礦工的密碼就好了

藍色框中的CPU關聯的部分，就是你再挖礦時要用多少CPU去挖(越多運算力就越強，但多的話電腦會明顯變慢)

補充→常見縮網址網站介紹及比較

大家好，我今天要把上次沒講完的縮網址給講完喔 如果你是第一次看到這篇文章，不知道啥麼是縮網址的話，可以按我前往了解什麼是縮網址。   那我們就進入正題吧，今天只講中文的喔，所以會講得很細喔<–還真詳細啊(酸   ppt.cc   成立四年的縮網址服務 為了讓這項服務能持續下去，所以會有一些限制，網址必須大於19個字元(包括前面的https://) 稍微講一下，「我同意使用規範」選項旁的「防堵機器人」選項，就是你在縮出網址之後給使用者點之後，會有驗證碼來請你輸入，之後才能導向你指定的網址 這是密碼保護的設定頁面 這個是你產生出來的縮網址頁面，按連結下方的[QR條碼]可以看到這個縮網址的QR code 這個是檔案保護，就是有點像密碼一樣，但和密碼不同的是，你必須上傳一個檔案(ppt.cc的大小限制是500kb)，使用這個必須持有你當初上傳的檔案，才能去到你指定的連結 原理的話，大概就是檔案的MD5吧(謎之聲:沒人問你這個，你幹嘛多嘴。小恩:我自己想說嘛) 這個是使用者點入加密的連結出現的授權頁面 如果是用密碼加密的話，點「輸入密碼」再把你設定的密碼打進去就好了 如果是用檔案加密的話，把當初上傳的檔案給它就好了 這個是解密完或是打完驗證碼之後的樣子，按一下連結就可以到目的網址了   以上，ppt.cc的說明到此結束   4fun.tw 這個了話我不知道要說甚麼了(謎之聲:那麼你可以滾了) 長度限制是19個字元(包含https://) 這個是安全設定，個人感覺「保存期限」和「安全門」的功能還蠻好用的 「保存期限」是指這個連結可以活多久，只要過了你設定的日期，這個連結就失效了 「安全密碼」就是使用者進入連結的時候，必須輸入密碼(這還需要說嗎?) 「安全門」就是縮網址只在你能從你設定的網址點擊，從其他來網頁點擊都會失效(謎之聲:怎們跟他們的網頁說明有些相似。小恩:巧合而已(汗))   這是連結產生後的樣子   這是一般沒經過任何設定的縮網址點開後的樣子，按一下左下角的[立即前往]就可以到達指定網址囉   這是有經過加密的縮網址，把密碼打下去之後按[讓我過!]就可以到指定的網址了   這東西叫做流量統計，怎麼使用呢 舉個例子：假如你縮出來的網址是 https://4fun.tw/1234 那麼就在就在網址後面加個「+」號，就像這樣 https://4fun.tw/1234+ 就可以了   今天的縮網址說明就到這囉，縮網址說明篇連載完畢QAQ   本文同步發布至合作部落格「無聊亂寫Blog」 作者部落格「墜落至IT的世界」 嚴重聲明:轉發須註明出處，否則一律視同盜文，作者有法律追溯權。
